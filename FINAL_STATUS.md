## Confirmed root cause

Three Base deployment scripts reused the hard-coded presale address `0xA7aa…b07C` after deploying a new AETH token. They excluded and funded that address without first checking `presale.token()`. The presale's immutable token reference points to `0xAb5a…9671e`, while the new Base AETH token is `0xecf7…8E4e`.

The unsafe scripts are disabled in PR #150. A guarded replacement script now deploys a fresh presale, checks chain ID, token ownership, constructor linkage, treasury linkage, tax exclusion, token inventory, and writes frontend configuration only after all checks pass.
