## Confirmed root cause

Multiple Base deployment/funding scripts reused the hard-coded presale address `0xA7aa…b07C` after deploying a new AETH token. They excluded and funded that address without first checking `presale.token()`. The presale's immutable token reference points to `0xAb5a…9671e`, while the new Base AETH token is `0xecf7…8E4e`.

The unsafe scripts are disabled in PR #150. The replacement script deploys a fresh presale and validates chain ID, token ownership, constructor linkage, treasury linkage, tax exclusion, and inventory before it writes frontend configuration.

## Safety actions completed

1. Disabled the public presale configuration on `main`.
2. Marked the recorded presale non-launchable in `smart-contract/deployments/presale-base.json`.
3. Recorded the invalid contract, linked token, 50,000,000 expected AETH balance, zero raised ETH, and cancellation/finalization state.
4. Added a guarded owner-only cancellation script for the invalid presale.
5. Added read-only Base verification and disposable fork testing.
6. Closed stale, non-mergeable Snyk PRs and recreated the OpenZeppelin fix from current `main`.
7. Disabled all discovered hard-coded token/presale deployment and funding scripts.
